Este análisis se realiza desde la posición de un atacante potencial y puede implicar la explotación activa de vulnerabilidades de seguridad. Tras la realización del ataque se presentará una evaluación de seguridad del sistema, indicando todos los problemas de seguridad detectados junto con una propuesta de mitigación o una solución técnica. La intención de una prueba de penetración es determinar la viabilidad de un ataque y el impacto en el negocio de un ataque exitoso. Sinónimo: Prueba de penetración

*[WAR]: Archivo Web de Java. Es un archivo que contiene una aplicación web Java. Los archivos WAR se utilizan para empaquetar y desplegar aplicaciones web Java en servidores de aplicaciones.

*[EAR]: Archivo de Aplicación Empresarial de Java. Es un archivo que contiene una aplicación empresarial Java. Los archivos EAR se utilizan para empaquetar y desplegar aplicaciones empresariales Java en servidores de aplicaciones.

*[Debugging]: Proceso de identificación y corrección de errores en el código de un programa. El debugging se realiza utilizando herramientas de depuración que permiten a los desarrolladores inspeccionar el código, ejecutarlo paso a paso y detectar los errores.

*[Testing]: Proceso de evaluación de la calidad y el rendimiento de un programa. El testing se realiza utilizando técnicas de pruebas que permiten a los desarrolladores identificar y corregir los errores, verificar el comportamiento del programa y garantizar su funcionamiento correcto.

*[IDE]: Entorno de Desarrollo Integrado. Es una herramienta que proporciona a los desarrolladores un conjunto de funcionalidades integradas para escribir, compilar, depurar y probar programas. Los IDEs suelen incluir un editor de código, un compilador, un depurador, un gestor de proyectos, etc.

*[UML]: Lenguaje de Modelado Unificado. Es un lenguaje estándar de modelado de software que se utiliza para visualizar, especificar, construir y documentar los sistemas de software. En UML, los desarrolladores pueden representar los componentes, las estructuras, los comportamientos, etc., de un sistema.

*[SCRUM]: Metodología ágil de gestión de proyectos que se basa en la iteración y la colaboración entre los miembros del equipo. En Scrum, los proyectos se dividen en iteraciones cortas llamadas sprints, en las que se planifican, se desarrollan y se entregan incrementos del producto.

*[CASE]: Ingeniería de Software Asistida por Computadora. Es un conjunto de herramientas y métodos que se utilizan para automatizar el proceso de desarrollo de software. Los CASEs incluyen herramientas de modelado, generación de código, gestión de proyectos, etc.

*[Hardcoding]: Práctica de incluir valores literales en el código fuente de un programa en lugar de utilizar variables o constantes. El hardcoding dificulta la reutilización, la mantenibilidad y la escalabilidad del código, ya que los valores están dispersos por todo el código.

*[Code Smell]: Indicador de un problema en el código fuente de un programa que puede afectar a su calidad, mantenibilidad o rendimiento. Los code smells suelen ser signos de malas prácticas de programación, como duplicidades, complejidad excesiva, acoplamiento fuerte, etc.

*[DRY]: Don't Repeat Yourself. Principio de diseño de software que se basa en la eliminación de duplicidades en el código fuente. El principio DRY promueve la reutilización del código, la modularidad y la mantenibilidad del software.

*[KISS]: Keep It Simple, Stupid. Principio de diseño de software que se basa en la simplicidad y la claridad del código fuente. El principio KISS promueve la legibilidad, la mantenibilidad y la eficiencia del software, evitando la complejidad innecesaria.

*[YAGNI]: You Aren't Gonna Need It. Principio de diseño de software que se basa en la eliminación de funcionalidades innecesarias en el código fuente. El principio YAGNI promueve la simplicidad, la eficiencia y la claridad del software, evitando la sobrecarga de funcionalidades.

*[SOLID]: Principios de diseño de software que se basan en la creación de código fuente limpio, mantenible y escalable. Los principios SOLID incluyen el Principio de Responsabilidad Única, el Principio de Abierto/Cerrado, el Principio de Sustitución de Liskov, el Principio de Segregación de Interfaces y el Principio de Inversión de Dependencias.

*[Agile]: Metodología de desarrollo de software que se basa en la iteración, la colaboración y la adaptación a los cambios. En Agile, los proyectos se dividen en iteraciones cortas llamadas sprints, en las que se planifican, se desarrollan y se entregan incrementos del producto.

*[OMT]: Técnica de Modelado de Objetos. Es una técnica de modelado de software que se utiliza para visualizar, especificar, construir y documentar los sistemas de software. En OMT, los desarrolladores pueden representar los objetos, las clases, las relaciones, etc., de un sistema.

*[RUP]: Proceso Unificado de Rational. Es un proceso de desarrollo de software que se basa en la iteración, la colaboración y la adaptación a los cambios. En RUP, los proyectos se dividen en fases, en las que se planifican, se desarrollan y se entregan incrementos del producto.

*[Lean]: Metodología de desarrollo de software que se basa en la eliminación de desperdicios y la mejora continua. En Lean, los proyectos se dividen en iteraciones cortas llamadas sprints, en las que se planifican, se desarrollan y se entregan incrementos del producto.
